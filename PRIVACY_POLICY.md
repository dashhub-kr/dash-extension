# DashHub 개인정보 처리방침
마지막 수정일: 2026-01-09

---

### **소개**

- 본 확장 프로그램 **"DashHub"**(이하 "확장")은 사용자가 알고리즘 문제 풀이 사이트(Baekjoon Online Judge, SW Expert Academy)에서 코드를 제출할 때, 제출한 코드와 문제를 자동으로 파싱하여 사용자의 GitHub 리포지토리에 커밋/보관할 수 있도록 도와주는 도구입니다.
- 본 문서는 확장이 수집·저장·처리하는 정보의 종류와 목적, 보관 방식 등을 투명하게 설명합니다.

---

### **수집 및 저장하는 정보**

본 확장은 별도의 중앙 서버를 운영하지 않으며, 모든 데이터는 사용자의 **로컬 브라우저 저장소(`chrome.storage.local`, `chrome.storage.sync`)** 와 사용자가 연결한 **GitHub 리포지토리**에만 저장됩니다.

#### **1. 인증 및 계정 정보**
- **저장 항목**:
  - `DashHub_token`: GitHub API 연동을 위한 개인 액세스 토큰 (OAuth)
  - `DashHub_username`: GitHub 사용자 ID
- **사용 목적**: GitHub API를 통해 리포지토리를 생성하거나 코드를 업로드(커밋)하기 위해 사용됩니다.

#### **2. 설정 및 상태 정보**
- **저장 항목**:
  - `DashHub_hook`: 코드를 저장할 타겟 GitHub 리포지토리 주소
  - `DashHubEnable`: 확장 프로그램 활성화/비활성화 상태
  - `mode_type`: 커밋 모드 설정 (자동/수동 등)
- **사용 목적**: 사용자의 환경 설정을 유지하고 확장이 올바르게 동작하도록 제어합니다.

#### **3. 문제 풀이 및 통계 데이터**
- **저장 항목**:
  - `stats`: 문제 풀이 내역 캐시
    - `submission`: 중복 커밋 방지를 위한 제출 파일의 SHA 해시 값
    - `problem`: 문제 제목, 난이도 등 메타데이터 캐시
    - `solvedac`: Solved.ac 티어 정보 캐시
- **사용 목적**: 이미 제출한 코드를 중복으로 업로드하지 않도록 확인하고, 문제 정보를 빠르게 불러오기 위해 로컬에 임시 저장합니다.

---

### **데이터 전송 및 외부 서비스**

본 확장은 기능을 수행하기 위해 다음의 외부 서비스와 통신합니다. **개발자의 별도 서버로 개인정보를 전송하지 않습니다.**

1.  **GitHub (API)**
    *   목적: 소스 코드 및 문제 정보 파일 업로드(커밋)
    *   전송 데이터: 소스 코드, 문제 설명(README), GitHub 토큰(인증용)
2.  **Solved.ac (API)**
    *   목적: 백준 문제의 난이도(티어) 정보 조회
    *   전송 데이터: 문제 번호 (익명 조회)
3.  **Baekjoon Online Judge / SW Expert Academy**
    *   목적: 사용자가 현재 보고 있는 문제 페이지와 제출 화면의 DOM 파싱
    *   방식: 사용자의 브라우저 내에서 스크립트가 로컬 실행되어 데이터를 추출합니다.

---

### **데이터의 보관 및 삭제**

- **보관 기간**: 확장을 사용하는 동안 사용자의 브라우저 내 로컬 스토리지에 저장됩니다.
- **삭제 방법**:
  - **확장 삭제**: 브라우저에서 확장을 삭제(제거)하면 로컬 스토리지(`chrome.storage.local`)의 모든 데이터는 즉시 삭제됩니다.
  - **데이터 초기화**: 확장 팝업 메뉴의 '연동 해제' 또는 설정 초기화 기능을 통해 GitHub 토큰 등 민감 정보를 삭제할 수 있습니다.
  - **GitHub 데이터**: GitHub 리포지토리에 업로드된 코드는 사용자가 해당 리포지토리에서 직접 관리/삭제해야 합니다.

---

### **권한 상세**

- `storage` / `unlimitedStorage`: 인증 토큰, 설정, 문제 풀이 캐시 데이터를 브라우저 내부에 저장하기 위해 사용합니다.
- `declarativeNetRequest`: 특정 사이트(알고리즘 사이트 등) 접근 시 올바른 동작을 위해 요청 헤더를 수정하는 데 사용될 수 있습니다.
- `host_permissions`: `github.com` (API 통신), `acmicpc.net`, `swexpertacademy.com` (코드 파싱), `solved.ac` (난이도 조회)에 접근하여 기능을 수행합니다.
